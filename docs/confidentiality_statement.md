# Confidentiality statement

This repository is designed as a public, confidentiality-safe research-software and engineering-data QA/QC project.

## Data boundary

The repository should not contain:

- real laboratory TDMS files;
- raw industrial or partner datasets;
- confidential facility paths;
- proprietary control logic;
- private acquisition settings;
- unpublished test reports;
- commercially sensitive operating conditions;
- personal data or operator information.

## Screenshots and figures

Screenshots, figures, and report examples should use either:

- synthetic data generated by this repository;
- anonymised file paths;
- anonymised channel/file names where needed;
- public-safe schematic examples.

Screenshots should not show real file-system paths, machine names, user names, partner names, raw experimental identifiers, or confidential test metadata.

## Synthetic TDMS examples

The synthetic TDMS generator is intended to imitate useful metadata patterns only. It may include examples such as:

- multiple groups;
- different sample rates;
- timestamp-like channels;
- CAN-style channel names;
- inactive or mostly constant channels;
- small group start-time offsets;
- intentionally awkward metadata cases.

It must not reproduce real measured signals or confidential operating conditions.

## What reviewers should infer

Reviewers may use this repository to assess:

- software structure;
- Python engineering-data handling;
- QA/QC workflow design;
- report generation;
- testing and validation awareness;
- public-safe communication of engineering workflows.

Reviewers should not infer that the synthetic examples represent a real facility, real operating envelope, real machine behaviour, or validated physical model.

## Maintainer checklist before publishing new examples

Before adding a screenshot, TDMS file, CSV, Excel file, or report to the repository, check that it contains no:

- real paths;
- private names;
- raw test IDs;
- partner names;
- real acquisition timestamps if sensitive;
- exact confidential sensor layouts;
- proprietary control logic;
- measured operational data.
